# 登录功能规范（v1.0）
## 1. 业务背景
用户通过用户名密码登录系统，获取访问令牌，用于后续接口鉴权。
## 2. 核心需求
- 支持用户名（6-20位）+ 密码（8位以上）校验
- 密码错误3次后，锁定账号10分钟
- 登录成功后，返回token并缓存至localStorage
- 登录失败时，返回明确的错误提示（如“用户名不存在”“密码错误”）
## 3. 边界条件
- 空用户名/密码：禁止提交，给出表单提示
- 网络异常：显示“网络错误，请重试”弹窗
- 已登录状态：跳转至首页，不显示登录页
## 4. 验收标准
- 用户名/密码格式校验符合接口规范
- 密码错误3次后，按钮置灰10分钟
- token 成功缓存，刷新页面后仍保持登录状态
